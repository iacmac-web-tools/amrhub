## Панель «Схемы»

Для того чтобы скрыть/показать панель расширенного поиска схем необходимо нажать на соответствующий значок.

Открывшаяся панель включается в себя:

  • Глобальный поиск по названиям схем (1)

    • Введите часть названия, чтобы быстро найти нужную схему среди доступных.

  • Поиск по тегам (2)

    • Выберите тег из выпадающего списка, чтобы отфильтровать схемы по тематике или назначению.

  • Создание нового тега (3)

    • Нажмите «+» рядом с полем тегов, чтобы создать новый тег и использовать его в будущем.

  • Фильтры по командам (4)
    • Схемы из моих команд — отображаются схемы, созданные вами и членами вашей команды.
    • Схемы из других команд — отображаются схемы, которыми поделились другие команды, при наличии доступа.

Для того чтобы просмотреть структуру схем, созданных вами, необходимо нажать на специальный значок.

Открывшееся окно отображает:

  • Глобальный поиск по названиям схем (1)

    • Введите часть названия, чтобы быстро найти нужную схему среди доступных.

  • Список схем (2)

    • Выберите схему из списка и нажмите кнопку "Выбрать", чтобы перейти к схеме

  • Кнопки действий (3)
    • Выбрать — осуществляеется переход к выбранной схеме.
    • Отмена — выход из окна просмотра структурной схемы.

Для того чтобы отфильтровать текущий список схем, используется локальный фильтр.

Для того чтобы пересортировать схемы в нужном вам порядке необходимо нажать на значок с точками (1); зажав левую кнопку мыши на значке с точками возле нужной схемы, перетащить её на необходимую позицию (2); нажать на значок галочки для сохранения результатов ручной сортировки (3).

## Карточка Схемы Схемы терапий представлены в виде списка карточек. Каждая карточка содержит следующую информацию:
  1. Название - краткое и понятное обозначение схемы.
  2. Описание - дополнительная информация о применении и особенностях схемы.
  3. Количество включённых схем - отображает, сколько дочерних схем входит в состав данной.
    4. Специальные значки, указывающие на наличие особых атрибутов:
    1. QR-код - схема доступна для просмотра по QR-коду.
    2. Тег - тематические метки, упрощающие поиск и фильтрацию.
    3. Команда - схема разработана или поддерживается определенной командой, либо доступна в рамках команды
    4. Сборник - схема входит в один или несколько сборников.
При нажатии на карточку схемы происходит следующее:
  • Выбранная схема подсвечивается.
  • Если у схемы есть дочерние схемы (1):
    • Справа отображается список этих дочерних схем.
    • Открывается подробная карточка выбранной схемы с полной информацией.
  • Если дочерних схем нет, то (2):
    • Просто открывается подробная карточка выбранной схемы без отображения дополнительного списка.
При нажатии на дочернюю карточку схемы реализовано следующее поведение:
  • Выбранная схема подсвечивается.
  • Если у выбранной дочерней схемы есть свои дочерние схемы (1):
    • Она перемещается влево, занимая центральную позицию.
    • Справа отображается список её дочерних схем.
  • Если у дочерней схемы нет собственных дочерних схем (2):
    • Она остается справа, а слева отображается её родительская схема.
    • При этом родительская схема визуально приглушена — отображается менее ярко, чтобы не отвлекать внимание от текущей.
В нижней части списка схем размещены кнопки для создания новых схем:
  1. Добавить основную схему — создаёт новую схему верхнего уровня, не связанную с текущими.
  2. Добавить дочернюю схему — добавляет схему в качестве дочерней к выделенной родительской (расположенной слева в интерфейсе).
## Панель работы с выбранной схемой Может содержать следующие элементы управления и информацию: 1. Название схемы 2. **Значок команды** - показывает, что схема принадлежит определенной команде. 3. **Кнопка QR-кода** - отображает QR-код и ссылку для просмотра схемы. 4. **Кнопка принтера** - позволяет сохранить схему в документ формата PDF для последующей печати. 5. **Кнопка полноэкранного просмотра** - раскрывает схему во весь экран для удобного просмотра и взаимодействия. 6. **Кнопка дополнительных действия** (троеточие) 7. **Переименовать** - позволяет изменить название и описание. 8. **Клонировать** - создает копию схемы. 9. **Редактировать содержимое** - дает возможность изменить содержание. 10. **Переместить схему** - перемещает схему в корень или к другой схеме. 11. **Скачать PDF** - позволяет скачать схему в PDF формате. 12. **Поделиться по ссылке/Сделать схему приватной** - при нажатии схема становиться публичной и ей можно поделить с помощью QR-кода или ссылки. 13. **Удалить** - удаляет текущую схему и все её дочерние. ## Карточка выбранной схемы Для удобной навигации в верхней части подробной карточки схемы отображается навигационная цепочка. Она показывает путь от корневой схемы к текущей. При клике на любой элемент цепочки можно быстро перейти к соответствующей схеме. ### Теги Ниже располагаются **теги** - тематические метки, которые помогают быстро находить схему через поиск и фильтры. Теги могут указывать на заболевание, тип терапии, целевую аудиторию и другие характеристики. ### Основные разделы Далее следуют основные разделы карточки: ### Текстовое описание схемы **Описание** — текстовое поле с дополнительной информацией о схеме (при использовании [Telegram-бота AMRnote](https://t.me/amrnote_bot) описание отображается в боте). **Содержание** — текстовое поле с основной информацией о схеме. ### Связанные схемы **Связанные схемы** — ссылки на другие схемы, имеющие логическую или тематическую связь. Содержат название и кнопки: 1. Перехода к схеме< 2. Предварительного просмотра (открывается в диалоговом окне) 3. Удаления связи ### Инфоблоки **Инфоблоки** — раскрывающиеся секции, содержащие текстовую информацию и прикреплённые файлы (документы, изображения и др.). ### Терапии **Терапии** — раскрывающиеся секции, содержащие препараты с необходимыми дозировками, а также описание терапии. ### Кнопка добавления блока Добавление новых блоков для схемы осуществляется с помощью раскрывающейся кнопки, которая включает следующие блоки. #### Добавление тега #### Добавление Инфоблока #### Добавление Терапии #### Добавление Дочерней схемы #### Добавление Связанной схемы
